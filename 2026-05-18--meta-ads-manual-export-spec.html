The report shape

Export from Meta Ads Manager at daily ad level. This preserves the timeline while still keeping campaign, ad set, and creative context.

01

Date range

Use last 60 days as the default. Use 30 days for fast triage; use 90 days for full account audit.

02

Row level

Rows should be grouped by Date + Campaign name + Ad set name + Ad name.

03

Metrics

Spend, impressions, reach, frequency, clicks, conversions, CPA, value, ROAS, statuses.

04

Attribution

Include or write down attribution setting/window. Without it, ROAS and CPA are ambiguous.

05

Export CSV

CSV is easiest. Weekly CSVs are acceptable only if they have identical columns.

One file or several reports?

Use one daily export unless Meta makes it impossible. Multiple weekly files are workable, but only if the columns match perfectly.

Best: one daily file

Last 60 days, ad-level, daily rows. Cleanest for trend diagnosis, fatigue detection, pauses, and current-vs-previous comparisons.

recommendedlowest error risk

Acceptable: weekly files

Several one-week CSVs are fine if every file has exactly the same columns and naming. More annoying, easier to screw up.

Weak: one week only

Useful for quick triage if spend and conversion volume are high. Too noisy for proper root-cause analysis in many accounts.

Breakdown Effect caveat

The manual package

If someone is exporting data for Hermes, ask for these in this order. Do not start with every possible breakdown; that creates noise.

Required

1. Daily ad-level performance

Rows: Date + Campaign + Ad set + Ad. Range: last 60 days. This is the main file.

Useful

2. Campaign/ad set summary

Rows: Campaign + Ad set. Range: last 30 days. Gives cleaner structure and budget overview.

Only if needed

3. Breakdown reports

Age, gender, placement, device, country/region. Use as clues, not blind instructions to cut spend.

Very useful

4. Tracking notes/screenshots

Attribution window, Pixel/CAPI status, optimized event, conversion definition, and value-passing status.

Reality check

5. Shopify/GA4/backend data

If available, use it to compare platform-reported conversions against actual revenue/leads.

Naming

6. Use clear filenames

meta_ads_daily_ad_level_YYYY-MM-DD_to_YYYY-MM-DD.csv keeps data provenance obvious.

Bottom line

The simple rule

Use one CSV with daily ad-level rows over 60 days. If the account is tiny, 90 days may be better. If the export gets enormous, start with 30 days plus a campaign/ad set summary.

Once MCP is connected, this manual export becomes a fallback. Until then, this is the highest-signal, lowest-chaos way to feed /skill meta-ads.
